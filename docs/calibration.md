# 区域与四点标定

## 处理顺序

固定顺序为：极坐标转平面 → FlipX/FlipY → Rotation → Offset → 距离/角度 → 有效多边形 → 四边死区 → 屏蔽多边形 → 聚类/跟踪 → Homography。改变顺序会让既有标定失效。

点云视图中可以拖动有效区顶点；屏蔽区可围绕当前目标新增并删除。边界点按有效区内处理，落入屏蔽区或四边死区的点被剔除。

## 四点标定

1. 固定雷达位置、朝向和安装变换。
2. 按界面顺序采集物理区域四角；四点必须形成不自交、非共线四边形。
3. Bridge 求解 3×3 Homography，将物理点映射到左下原点的标准矩形。
4. 保存前检查四角误差；无效或退化输入会被拒绝。
5. 修改型号、Flip、Rotation 或 Offset 后重新标定。配置会保存型号、安装变换快照、矩阵、时间和最大角点误差。

## 现场验收

- 四角、中心和四边中点均能稳定到达 Unity 对应位置。
- 边缘停留时没有持续跳变；必要时先微调四边死区，再调跟踪平滑。
- 两个目标交叉前后 TrackId 不应无理由频繁交换。
- 保存配置、重启 Bridge 后复测相同点位。
